Querydsl (Dynamic Query & Paging)

Querydsl을 적용하여 국가, 수도, 대륙 같은 다양한 조건의 동적 쿼리를 타입 안전성이 보장된 코드로 구현하였습니다.
추가로 Pageable 인터페이스를 활용해 복잡한 limit과 offset 계산을 자동화하여 효율적으로 페이징 처리를 하였습니다.

AdmNationRepositoryImpl.jave 

public class AdmNationRepositoryImpl implements AdmRepositoryCustom {
    · · · 중략 · · ·
    @Override
    public Page<AdmNationVO> search(AdmSearchCond cond, Pageable pageable) {
        QAdmImageVO image = QAdmImageVO.admImageVO;
        List<AdmNationVO> content = queryFactory
            .selectFrom(admNationVO)
            .where(searchCond(cond.getSearch()))
            .orderBy(admNationVO.regDate.desc())
            .offset(pageable.getOffset())
            .limit(pageable.getPageSize())
            .fetch();

        Long total = queryFactory
            .select(admNationVO.count())
            .from(admNationVO)
            .where(searchCond(cond.getSearch()))
            .fetchOne();

        return new PageImpl<>(content, pageable, total != null ? total : 0L);
    }

    private Predicate searchCond(String keyword) {
        if(keyword == null || keyword.isBlank()) {
            return null;
        }
        return admNationVO.nationNameKo.contains(keyword)
            .or(admNationVO.nationNameEn.contains(keyword))
            .or(admNationVO.capitalKo.contains(keyword))
            .or(admNationVO.capitalEn.contains(keyword));
    }
}

트러블슈팅

1. N + 1 문제

상황

목록 화면을 조회할 때 이미지 정보를 가져오기 위해 이미지 개수만큼 추가 쿼리가 발생하는 N + 1 문제를 발견하였습니다.

분석

불필요한 조인 쿼리를 방지하기 위해 지연로딩으로 설정했으나 목록 조회 과정에서 자식 엔티티에 접근할 때 메인 쿼리 외에 리스트의 개수만큼 자식 테이블 조회 쿼리가 추가로 발생하게 되었습니다.

1차 시도

Fetch Join을 적용해 보았으나 해결되지 않았고 Fetch Join과 페이징 기능을 함께 쓸 수 없다는 것을 알게되었습니다.

해결

application.properties에서 default_batch_fetch_size=100 으로 전역 설정을 하여 페이징을 유지하면서 쿼리 실행 횟수를 1/100 수준으로 최적화하였습니다.

2. Dirty Checking(변경 감지) 이슈

상황

상세보기 화면을 조회할 때 의도하지 않은 update 쿼리가 발생하는 Dirty Checking를 발견하였습니다.

분석

DTO 생성자 내부에서 관습적으로 사용하던 Setter 메서드가 JPA의 영속성 컨텍스트에 의해 엔티티 변경으로 간주되어 Dirty Checking이 발생하게 되었습니다.
추가로 분석해보니 Setter 메서드를 호출하는 방식은 객체 불변성을 해칠 수 있다는 것을 알게되었습니다.

해결

Setter 메서드를 제거하고 필드 직접 초기화 방식으로 변경하여 Dirty Checking 이슈를 해결하고 객체 무결성을 확보하였습니다.
